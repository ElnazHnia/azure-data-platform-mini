# 📊 P&L Forecast Dashboard – Azure Data Platform

## 📌 Project Overview

This project implements an end-to-end cloud-based data platform in Microsoft Azure to generate and visualize a Profit & Loss (P&L) forecast dashboard.

The solution follows modern Data Engineering and DevOps best practices, including Infrastructure-as-Code and CI/CD automation using GitHub Actions. The entire architecture was built directly in Azure and connected to a structured Git-based deployment workflow.

---

## 🏗️ Architecture

The platform is fully deployed in Microsoft Azure and consists of the following components:

- Azure Data Factory (ADF) – Data orchestration and transformation  
- Azure SQL Database – Structured relational storage  
- Azure Storage Account – Raw/staging data layer  
- Power BI – Business intelligence and dashboard visualization  
- GitHub Actions – CI/CD automation  
- ARM Templates – Infrastructure as Code deployment  

This architecture enables automated, reproducible deployments and scalable cloud data processing.

---

## 🔄 Data Flow

1. Raw data is stored in Azure Storage.  
2. Azure Data Factory pipelines:
   - Extract raw data  
   - Transform and clean datasets  
   - Load structured data into Azure SQL Database  
3. Power BI connects to the SQL Database.  
4. The dashboard visualizes P&L forecast metrics and financial KPIs.  

---

## 🚀 CI/CD Implementation

The project implements a complete CI/CD workflow using GitHub Actions.

### Continuous Integration (CI)

Triggered on push or pull request to the main branch:

- Validates repository structure  
- Checks JSON syntax of ADF artifacts  
- Ensures configuration consistency  

This ensures quality control before deployment.

### Continuous Deployment (CD)

Triggered when:
- Manual workflow dispatch, or  
- A push to the adf_publish branch (after ADF Publish)

Deployment process:

1. Checkout of adf_publish branch (contains ARM templates generated by ADF)  
2. Azure login using a Service Principal  
3. ARM template deployment via Azure CLI  
4. Automatic update of Azure Data Factory  

All infrastructure changes are deployed via ARM templates, ensuring Infrastructure-as-Code principles.

Deployment authentication is handled via:

- Azure Service Principal

## 🎯 DevOps Workflow

1. Develop and modify pipelines in ADF Studio  
2. Commit changes to main  
3. Publish from ADF → updates adf_publish  
4. GitHub Actions automatically deploys to Azure  
5. Azure Resource Group shows successful ARM deployment  

This creates a complete Dev → Publish → Deploy pipeline.
